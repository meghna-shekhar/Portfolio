Project

Personalized Story Book Generation with Fine-tuned LLM

Large Language Models · Multimodal AI · Education

Overview

This project builds a cost-efficient multimodal system that generates personalized storybooks for early learners (K–2). It fine-tunes an open-source large language model using UFLI-inspired learning materials and combines it with a custom computer vision pipeline to produce consistent, developmentally appropriate text and illustrations.

All stories, images, and vocabulary are stored in Supabase, enabling classroom deployment, progress tracking, and teacher-in-the-loop refinement.

Motivation

Many AI story generators ignore pedagogy: reading level, phonics progression, and curriculum alignment are often an afterthought. In this project, the goal is to align LLM-generated stories with K–2 literacy goals while keeping the system practical for schools in terms of cost and tooling.

System design

The system is organized into three main components:

  • LLM story engine: fine-tuned open-source model that conditions on grade level, target phonics pattern, vocabulary list, and reading goals to generate age-appropriate narratives.
  • Vision & illustration pipeline: prompts an image model to create consistent characters and settings across pages, keeping visual style coherent throughout a book.
  • Data layer (Supabase): stores books, pages, illustrations, vocabulary, comprehension questions, and per-student usage logs.

Personalization flow

  1. Teacher selects a student profile (grade, reading level, interests, goals).
  2. The system constructs a story plan (setting, characters, plot beats, targeted phonics patterns).
  3. The LLM generates multi-page text aligned with those constraints and injects controlled vocabulary for practice.
  4. For each page, the vision pipeline generates an illustration that reuses the same characters and visual style.
  5. Generated assets are saved to Supabase and surfaced through a dashboard and student-facing reading interface.

Impact

  • Enables teachers to quickly produce custom storybooks matched to individual students and small groups.
  • Maintains pedagogical alignment with UFLI-style literacy progressions rather than generic storytelling.
  • Provides a foundation for future experimentation with multimodal feedback and long-term progress tracking.

Tech stack

Languages: Python, TypeScript
AI: Open-source LLM (fine-tuned), image generation model, prompt engineering for consistency
Backend: Supabase (PostgreSQL + storage), REST/JSON APIs
Topics: Multimodal AI, education technology, model fine-tuning, teacher-in-the-loop design
