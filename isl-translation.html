Project

Indian Sign Language to MOCAP

Deep Learning · Pose Estimation · Assistive Technology

Overview

This project builds a translation pipeline that converts text or speech into Indian Sign Language (ISL) motion. Starting from ISL educational videos, I estimate 2D human pose, clean and normalize the joints, and export the result as MOCAP-style keypoint sequences that can drive a 3D character.

Motivation

Most sign language technologies focus on ASL, and they usually assume controlled recording conditions. ISL users often rely on human interpreters, which limits accessibility. I wanted to explore whether modern pose estimation can help build a bridge between spoken/text language and ISL, especially under realistic lighting, occlusion, and camera angles.

What the system does

At a high level, the system takes an input phrase, maps it to a sequence of ISL glosses, and then plays the corresponding motion on a character rig:

  1. Pose extraction: run 2D human pose estimation on ISL teaching videos to obtain joint coordinates for each frame.
  2. Cleaning & normalization: fix missing joints, smooth jitter, and normalize skeletons so that sequences can be compared and reused.
  3. Motion library: store cleaned pose sequences as reusable “building blocks” for individual signs.
  4. Text–to–sign mapping: map English phrases to ISL sign order and stitch together the corresponding motion clips.
  5. Export to MOCAP: write the final joint trajectories in a format suitable for rendering with a 3D character.

Technical highlights

  • Implemented the full pipeline in Python using TensorFlow, Keras, OpenCV, and NumPy.
  • Designed post-processing to handle occluded or missing joints and keep motion physically plausible.
  • Experimented with different normalization strategies to make pose sequences reusable across videos and subjects.
  • Structured the data so it can plug into downstream 3D rendering tools or real-time animation systems.

Impact and learnings

The prototype demonstrates that we can go from raw ISL videos to reusable motion segments that approximate sign-language gestures. It also surfaced practical challenges: noisy landmarks, camera angles that obscure hands, and linguistic nuances when mapping English word order to ISL.

From an engineering perspective, the project taught me how to design an end-to-end vision pipeline, debug pose-estimation failures, and think about how ML systems support real accessibility use cases rather than just benchmarks.
