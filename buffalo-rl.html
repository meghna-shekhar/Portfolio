Project

Buffalo Accident Risk Prediction & Resource Allocation using Reinforcement Learning

Reinforcement Learning · City Analytics · Geospatial Data

Overview

This project builds a Reinforcement Learning (RL) system to help a city allocate emergency resources (ambulances, fire trucks, etc.) more effectively. Using real-world traffic and accident data from Buffalo, NY, the agent learns which zones are high risk and how to position limited vehicles to reduce response time.

Problem

Emergency services must decide where to station vehicles before incidents occur. Historically, this is based on simple heuristics or static hot-spot maps. We wanted to see whether RL could:

  • Predict high-risk regions in the city from historical patterns.
  • Continuously adapt resource placement as conditions change.
  • Outperform a random or naive allocation strategy.

Data & environment

We processed Buffalo accident incident data by location, type, and time, then aggregated it into:

  • Zip-code level heatmaps.
  • Finer 500 m grid-cell risk scores.

These maps define a custom RL environment where each state describes current risk in different zones and where emergency vehicles are stationed.

Approach

We used a Proximal Policy Optimization (PPO) agent in an Actor–Critic architecture:

  • Actor and critic implemented as multi-layer perceptrons.
  • PPO with clipped objective for stable policy updates.
  • Reward shaped around response time and coverage of high-risk zones.
  • Baseline: random placement of vehicles across city zones.

Results

Over training, the PPO agent learned to focus coverage on high-risk areas, improving long-term reward compared to the random baseline. We visualized accident-prone regions and agent behavior via:

  • City-wide risk maps (ZIP and 500 m grid level).
  • Side-by-side comparisons of PPO vs. random allocation.

Key learnings

  • How to translate messy, real-world geospatial data into a well-defined RL environment.
  • Practical tricks for stabilizing PPO training (reward scaling, clipping ranges, and baselines).
  • The importance of visualization for explaining RL policies to non-technical stakeholders.

Tech stack

Languages: Python
Libraries: stable-baselines / PPO implementation, NumPy, Pandas, Matplotlib, GeoPandas
Topics: Reinforcement Learning, PPO, City-scale analytics, geospatial visualization
